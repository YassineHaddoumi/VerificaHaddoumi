1) Le commit create in una repository locale non vengono sincronizzate
automaticamente con la repository online collegata.

2) Il nuovo nome standard del branch principale utilizzato come
convenzione in GIT è main

3) Il workflow dei comandi di GIT è git add -> git commit -> git push

4)Il comando da utilizzare per vedere l’elenco di tutte le commit del
branch in cui ci si trova e i loro hashcode è git log

5)Il comando per effettuare il rollback ad una commit precedente è git reset  --hard

6)Due motivazioni per giustificare l’utilizzo di GIT: 1)un gruppo di collaboratori puo lavorare su dei file di uno stesso progetto; 2)si puo lavorare su un progetto facendo un branch 
senza modificare il branch principale,  se va bene puoi aggiungere un file al branch principale altrimenti puoi lasciarlo stare o eliminare

7) Git clone serve a clonare cioe copiare una repository online su un pc in locale, pero la cartella dove devo inserire le la repository non deve essere ancora inizializzata 
Git pull serve a prendere i file dalla repository
